# MovieRatingDatabase

## Project Objective
The objective of this project is to build and explore a movie rating system using SQL to understand how users rate movies, 
identify popular movie titles, and perform meaningful analysis using SQL statements.

# Dataset
The dataset used in this project consists of three main tables:

## Users Table
This table stores information about users who rate movies. The fields include:
user_id
name
age
gender
location

## Movies Table
This table contains information about movies. The fields include:
movie_id
title
genre
release_year

## Ratings Table
This table records the ratings given by users to different movies. The fields include:
user_id
movie_id
rating_value
rating_date
These tables are connected using user_id and movie_id to create relationships between users, movies, and ratings.

# Process
The following steps were carried out in completing this project:

📌Creation of a database named MovieRatingDB.
📌Creation of three tables: Users, Movies, and Ratings.
📌Insertion of sample data into the tables.
📌Execution of SQL queries to retrieve and analyze the data.

# Analysis
SQL queries were used to analyze the data in the database. The analysis focused on identifying highly rated movies, 
determining the average ratings of movies, and examining patterns in how users rate movies.

# Conclusion
This project demonstrates how SQL can be used to build and manage a relational database.
It also shows how data stored in tables can be analyzed using SQL queries to gain useful insights about movie ratings
and user preferences.
